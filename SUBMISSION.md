# تسليم واجب المعسكر — نظام وكلاء الذكاء الاصطناعي

> **الاسم:** Threat Intel & Asset Monitor
> **النوع:** نظام متعدد الوكلاء (Multi-Agent System) عبر MCP Connectors
> **الشق المطبّق:** الشق الثاني — Golden Dataset لاختبار أمان الوكلاء

---

## 1. فكرة المشروع

نظام أمني متعدد الوكلاء يؤتمت رصد الثغرات في البرامج المثبتة على الجهاز.
بدل المراجعة اليدوية لكل برنامج، أربعة وكلاء يتعاونون تحت وكيل مدير
(Orchestrator) لإنجاز المهمة تلقائياً، باستخدام أدوات MCP للوصول للملفات
المحلية والبحث في الإنترنت.

**المشكلة المحلولة:** متابعة الثغرات يدوياً بطيئة ومملة — النظام يسوّيها بضغطة زر.

---

## 2. الهيكلة (Architecture)

```
              👤 المستخدم
                  │
            🧭 Orchestrator  (الوكيل المدير — ينسّق التتابع)
                  │
     ┌────────────┼────────────┐
     ▼            ▼            ▼
🗂️ Asset      🛰️ Threat     📝 Remediation
 Auditor        Hunter        (يكتب التقرير)
(قراءة فقط)   (شبكة فقط)      (كتابة مسار ثابت)
     │            │            │
 inventory.txt   NVD API   threat_intel_report.md

            + 🔄 Package Manager (يفحص/يطبّق تحديثات winget)
```

| الوكيل | الدور | الصلاحية (أقل امتياز) |
|---|---|---|
| Orchestrator | تنسيق التتابع وجمع النتائج | تنسيق فقط |
| Asset Auditor | قراءة البرامج المثبتة | قراءة فقط، داخل الصندوق |
| Threat Hunter | بحث CVE في NVD | شبكة فقط، مضيف NVD ثابت |
| Remediation | كتابة التقرير النهائي | كتابة مسار واحد ثابت |
| Package Manager | فحص/تطبيق تحديثات winget | argv ثابت، بلا shell |

---

## 3. أدوات الـ MCP المستخدمة

1. **Desktop Commander (Filesystem MCP)** — قراءة/كتابة الملفات.
2. **أداة البحث المدمجة (Web Search)** — جلب ثغرات CVE من NVD.

---

## 4. الوظائف

- فحص أمني متعدد الوكلاء ينتج تقريراً حقيقياً بثغرات مطابقة لإصدارات الجهاز.
- فحص وتطبيق تحديثات البرامج تلقائياً عبر winget.
- واجهة ثنائية اللغة (عربي/إنجليزي) بدعم RTL/LTR كامل.
- ملخص سريع للتقرير + تصميم واجهة احترافي بأيقونة أصلية.
- تطبيق سطح مكتب مستقل + حزمة مشاركة آمنة (source + SHA-256).

## 5. الشق الثاني — Golden Dataset لاختبار أمان الوكلاء

مجموعة اختبارات **حقيقية تُنفَّذ ضد كود النظام فعلياً** (ليست أمثلة ثابتة)،
تغطي 19 فئة هجوم: الأربع المطلوبة + 15 إضافية لتغطية كامل سطح الهجوم (منها حماية LLM من حقن التعليمات، تشفير الأسرار DPAPI، سلسلة تدقيق مقاومة للتلاعب، تحديد معدّل الطلبات، أمان وكيل القرار، وقفل مستلم البريد).

**النتيجة: 59 اختبار · 59 ناجح · 0 فاشل · 100%**

| # | نوع الاختبار | وصف الاختبار | النتيجة المتوقعة | آلية الحماية |
|---|---|---|---|---|
| 1 | Prompt Injection | إدخال ملوّث في inventory.txt (metachar/injection/unicode) | رفض السطر | قائمة أحرف مسموح بها + حد طول |
| 2 | Data Exfiltration | الخروج من صندوق الملفات (traversal/absolute) | SecurityError | resolve() + تحقق داخل مجلد العمل |
| 3 | Jailbreak | تجاوز CSRF/Host (forged host, no token, cross-origin) | 403 / 421 | Host allowlist + توكن CSRF سري |
| 4 | Tool Misuse | تثبيت حزمة winget غير مفحوصة | رفض العملية | معرّفات من فحص للقراءة فقط + regex |
| 5 | SSRF / Egress | تحويل وجهة البحث لمضيف خارجي | يبقى NVD | مضيف ثابت + urlencode |
| 6 | DoS | Content-Length كاذب/سالب، ملف ضخم | رفض | حدود حجم للجسم والملف |
| 7 | Stored XSS | نص CVE خبيث يصل الواجهة | لا تنفيذ | CSP صارمة + escapeHtml |
| 8 | State & Concurrency | فتح نسختين / سباق حالة | منع | قفل PID للنسخة الواحدة |
| 9 | Least Privilege | تجاوز الوكيل صلاحيته | منع | فصل قدرات قراءة/شبكة/كتابة |
| 10 | URL Injection | حقن بارامتر في استعلام NVD | لا يتغير الهيكل | urlencode للمدخلات |

## 5.1 ميزات الذكاء الاصطناعي والأتمتة (النسخة النهائية)

أُضيفت لتقوية "الذكاء الاصطناعي جزء أساسي من الحل" في متطلبات المشروع النهائي:

- **شرح ذكي (LLM)**: زر "اشرح" بجانب كل ثغرة يشرحها بعربي مبسّط لمستخدم غير تقني.
- **إعادة تقييم ذكية (LLM)**: وكيل يفلتر نتائج NVD ويحدد أيها فعلاً ذو صلة (يقلّل الإنذارات الخاطئة القديمة) — قرار agentic حقيقي.
- **سؤال وجواب طبيعي (LLM)**: تسأل عن تقريرك بلغتك ويجاوب من بياناته.
- **مقارنة الفحوصات**: يعرض الثغرات الجديدة والثغرات التي انحلّت منذ آخر فحص.
- **تاريخ المخاطر**: رسم بياني لعدد الثغرات عبر الزمن.
- **فحص دوري تلقائي + بريد**: يفحص كل (3 أيام / أسبوع / أسبوعين / شهر) ويرسل التقرير لبريدك تلقائياً.

**النموذج**: محلي عبر **Ollama** (مجاني، بلا تكلفة، بلا إنترنت) — يطابق قاعدة المشروع "AI فعلي".

### إعداد الميزات (اختياري — التطبيق يعمل بدونها)
1. الذكاء: ثبّت [Ollama](https://ollama.com) ثم `ollama pull llama3.2`.
2. البريد: انسخ `app/config.local.example.json` إلى `config.local.json`، و`app/secrets.local.example.json` إلى `secrets.local.json`، وعبّئ بيانات SMTP وكلمة المرور بنفسك.

### قيود الأمان لهذه الميزات
- بيانات CVE (غير موثوقة) تُغلَّف في كتلة معزولة داخل نداء LLM، ورسالة النظام تمنع تنفيذها كتعليمات (حماية Prompt Injection).
- الأسرار في `secrets.local.json` فقط — لا تظهر بأي رد API ولا بأي سجل، ومستبعدة من نسخة المشاركة.
- البريد **send-only** ومقفول على عنوانك المُعدّ فقط — لا يمكن لأي مدخل تغيير المستلم.
- كل نقاط API الجديدة محميّة CSRF، ونداءات LLM محدودة المعدّل.

## 5.2 وكيل القرار + مؤشر الصحة الأمنية + مطابقة الاستغلال الفعلي

ثلاث إضافات تحوّل النظام من "يبلّغ بالنتائج" إلى "يتخذ قراراً ويشرح سببه":

- **KEV Checker Agent**: يقارن كل CVE بقائمة CISA الرسمية للثغرات المستغلة فعلياً بالعالم الحقيقي (Known Exploited Vulnerabilities) — يميّز "خطر نظري" عن "خطر مؤكد الاستغلال".
- **Decision Agent**: يصنّف كل نتيجة إلى (عاجلة / روتينية / معلوماتية) بناءً على الخطورة + الاستغلال الفعلي، ويحسب **مؤشر صحة أمنية** (0-100) لكامل الجهاز.
- **قيد الأمان الصارم (بناء على طلبك)**: وكيل القرار **لا يملك أي صلاحية تنفيذ** — لا يستورد `package_manager` ولا `subprocess` إطلاقاً (مُتحقَّق آلياً عبر تحليل AST بالكود، مو بس وعد). إذا صنّف عنصراً "عاجل"، يظهر **تنبيه بارز** بالواجهة بزرّين فقط: **"اذهب للتحديث"** (يوصلك لتبويب التحديثات الآمن الموجود مسبقاً، بلا تنفيذ تلقائي) أو **"أجّل"** (غداً / 3 أيام / أسبوع) — والتأجيل نفسه بيانات توقيت فقط، بلا أي قدرة تنفيذية. القرار النهائي دائماً لك.

## 5.3 إعادة بناء الواجهة (SOC Console)

واجهة جديدة كاملة من الصفر: تخطيط شريط جانبي (بدل التبويبات العلوية)، شريط مؤشرات KPI (مؤشر الصحة، إجمالي النتائج، الحرجة/العالية، الأصول)، لوحة تنبيهات عاجلة، وتصميم داكن مريح للعين (لون واحد تفاعلي + ألوان خطورة دلالية فقط — بلا توهّج أو حركة مزعجة). مبني بمساعدة أدوات تصميم متخصصة (`ui-ux-pro-max`) واستطلاع لأفضل ممارسات واجهات مراكز العمليات الأمنية (SOC dashboards).
